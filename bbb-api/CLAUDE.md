- When writing unit tests for the bbb-api the testing functions should be implemented in a Go table testing style. Only use Go standard library features. Testing/assertion libraries and frameworks should never be used. The goal should be to achieve 100% code coverage on all code paths being tested by the function. Unit tests should be placed in a new file in the same directory as the function being tested. The new file should have the same name as the file where the function resides with "_test" appended to the name before the ".go" extension.
